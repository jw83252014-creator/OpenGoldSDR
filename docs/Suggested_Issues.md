# Suggested GitHub Issues - OpenGoldSDR

## 1. Reality check: SDR vs EM vs GPR for prospecting

Review whether SDR hardware can provide useful field data for mineral prospecting, or whether the project should shift toward EM induction, magnetometer, or GPR-style hardware.

Labels: research, feasibility, hardware

## 2. Define minimum useful field-test rig

Propose the cheapest useful test setup for real-world experiments.

Include:

- Hardware list
- Expected cost
- Frequencies or sensing method
- Antenna / coil notes
- Power requirements
- Data logging method
- Phone/laptop interface

Labels: hardware, field-testing

## 3. Build Georgia Gold Belt public dataset list

Collect useful public data sources for Georgia Gold Belt research.

Possible categories:

- Historical mine locations
- USGS mineral occurrence data
- State geology maps
- Topographic data
- LIDAR / DEM terrain data
- Creek and drainage data
- Soil / mineralization maps

Labels: data, geology, Georgia

## 4. Design repeatable GPS-tagged field log format

Create a field log template for repeatable tests.

Should include:

- GPS point
- Date/time
- Weather
- Soil moisture estimate
- Terrain type
- Nearby quartz / iron staining / old workings
- Hardware used
- Raw signal files
- Notes and photos

Labels: field-testing, documentation

## 5. Create first signal-processing notebook

Add a starter notebook/script for importing field signal data and plotting basic results.

Labels: software, signal-processing

## 6. Add safety and legal notes

Add prospecting safety and permission reminders.

Include:

- Land access
- Digging permission
- Protected sites
- Mine hazards
- Electrical/radio safety
- Environmental care

Labels: documentation, safety

## 7. Pick an open-source license

Choose an open-source license before accepting outside code contributions.

Labels: repo-maintenance, legal
